# Demo Notes

## What to show

- complaint creation from the dashboard
- triage review flow
- complaint promotion to the board
- board movement across workflow stages
- seeded member and label data

## Suggested explanation

1. Complaints first appear in the intake workflow.
2. Triage is used to validate and classify them.
3. Accepted complaints move to the active board.
4. The board tracks progress from backlog to completion.

## Teacher-facing points

- the project has a structured local setup flow
- complaints are persisted through Prisma and PostgreSQL
- the dashboard supports both intake review and workflow tracking
- the commit history shows migration, setup, branding, and workflow fixes
